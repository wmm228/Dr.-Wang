# 博客系统使用指南

这个文档将指导你如何添加新的博客文章，并让它显示在“春、夏、秋、冬”的分类中。

## 1. 准备博客文章

1.  使用任何 Markdown 编辑器（如 Typora, VS Code, Obsidian）编写你的博客。
2.  将编写好的 `.md` 文件保存到项目的 `posts/` 文件夹下。
    *   例如：`posts/my-new-trip.md`
3.  如果文章中有图片，建议将图片放在 `static/img/blog/` 目录下（如果没有该目录可以新建），然后在 Markdown 中这样引用：
    *   `![图片描述](../static/img/blog/my-photo.jpg)`

## 2. 注册博客文章 (最重要的一步)

前端网页无法自动扫描文件夹，所以每写一篇新文章，你都需要在 `static/data/posts.json` 文件中“注册”它。

打开 `static/data/posts.json`，你会看到一个列表，参考下面的格式添加一项：

```json
{
    "id": "unique-id-001",              // [1] 唯一ID，不能重复，可以用数字或英文
    "title": "这里写标题",               // [2] 文章标题
    "date": "2024-05-20",               // [3] 发布日期
    "category": "spring",               // [4] 分类：spring(春), summer(夏), autumn(秋), winter(冬)
    "description": "这里写简短的介绍...", // [5] 显示在卡片上的简介
    "cover": "./static/img/cover.jpg",  // [6] 封面图片路径
    "file": "./posts/my-new-trip.md",   // [7] 你的 Markdown 文件路径
    "tags": ["旅行", "生活"]             // [8] 标签
}
```

### ⚠️ 注意事项：
*   **ID** 必须唯一。
*   **category** 只能是 `spring`, `summer`, `autumn`, `winter` 其中之一，否则无法在分类页显示（但会在“全部”中显示）。
*   **JSON 格式** 很严格，最后一项后面**不能**有逗号 `,`，每一项之间**必须**有逗号。如果不小心写错了，网页可能会加载失败。

## 3. 查看效果

保存所有文件后，刷新网页，点击导航栏的“博客”，你应该能看到新添加的文章了。
点击卡片，即可进入阅读模式。

## 4. 高级功能

*   **数学公式**：支持 LaTeX 语法，如 `$E=mc^2$`。
*   **代码高亮**：支持 ```python ... ``` 等代码块。
